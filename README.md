# 🏦 AI Bank Consultant: RAG-система для банковской сферы

## 📌 Описание проекта
Данный проект представляет собой разработку вопросно-ответной системы (RAG — Retrieval-Augmented Generation) для консультирования клиентов банка. Система способна анализировать банковскую документацию (кредиты, вклады, ипотека, FAQ) и предоставлять точные ответы, опираясь исключительно на факты из базы знаний, что критически важно для предотвращения галлюцинаций LLM в финансовом секторе.

## 🛠 Технологический стек
* **Оркестрация:** LangChain
* **Векторная база данных:** FAISS
* **Модель эмбеддингов:** `intfloat/multilingual-e5-large` (выбрана за высокое качество работы с русским языком)
* **Генеративная модель (LLM):** `Qwen/Qwen2.5-1.5B-Instruct`
* **Дополнительно:** Rank-BM25 (для гибридного поиска), HuggingFace Transformers

## 📂 Структура репозитория
* `/NLP_II/` — синтетический набор данных из 5 документов (условия кредитования, вклады, ипотека, требования к заемщикам, FAQ).
* `NLP_II_Project_Kozhenkova.ipynb` — Jupyter Notebook с полным пайплайном данных, ретривера и LLM.
* `README.md` — описание проекта и выводы.

## 📊 Этапы реализации и архитектурные выводы

### Этап 1. Подготовка данных и эмбеддинги
Документы были очищены и приведены к единому формату. Для разбиения текста применялся рекурсивный чанкинг (RecursiveCharacterTextSplitter), что позволило сохранить смысловую целостность абзацев. Векторизация выполнена с помощью модели `multilingual-e5-large`, а в качестве легковесного и быстрого хранилища выбрана in-memory база FAISS.

### Этап 2. Система ретрива (Поиск)
Был реализован и протестирован кастомный пайплайн поиска:
1. **Гибридный поиск:** Объединение векторного поиска (поиск по смыслу) и алгоритма BM25 (поиск по ключевым словам).
2. **Контекстное сжатие:** Внедрен фильтр на основе L2-дистанции (порог отсечения `0.4`), который безжалостно отсекает информационный шум, оставляя только релевантные факты.
3. **Метрики качества:** На синтетическом датасете из 20 вопросов система показала высочайшую точность:
   * **Hit Rate@3:** 0.95 (95% запросов находят нужный файл)
   * **MRR:** 0.900

### Этап 3. Интеграция с LLM и Prompt Engineering
В систему интегрирована локальная модель Qwen 1.5B. В ходе экспериментов был выявлен классический RAG trade-off: 
Смягчение системного промпта и повышение температуры (temp=0.3) приводили к появлению галлюцинаций (модель начала выдумывать этапы получения ипотеки), идеальный баланс был найден при снижении температуры до 0.1 и контролируемом по жесткости промпте. 
**Решение:** Для банковского домена был внедрен ограничивающий промпт («Отвечай на основе текста») и околонулевая температура, что обеспечило строгую привязку к документам. Также реализована память диалога (Memory) для понимания контекста и прозрачное цитирование источников.

### Этап 4. Оптимизация производительности
Внедрение механизма in-memory кеширования продемонстрировало феноменальную эффективность. Время генерации ответа LLM «с нуля» составило ~9.54 сек., в то время как выдача готового ответа из кеша заняла всего 0.000316 сек. Итоговое **ускорение системы при повторных запросах составило более 3000 раз**, что позволяет масштабировать решение для работы под высокой нагрузкой.
